## 목적

문서 구조 정리를 마친 뒤, 실제 게임 구현에 바로 연결되는 수준으로 규칙을 고정한다.

---

## 0) 진행 원칙

- 당분간 레포 커밋/푸시는 진행하지 않는다.
- 설정은 서사 문장으로 설명하되, 각 항목마다 최소 1개의 판정 기준을 함께 둔다.
- 추상적 표현은 유지하되, "언제/무엇으로/어떻게"가 보이도록 작성한다.

---

## 1) 정식 스펙 A — 퇴장 절차 실패의 3단계화

퇴장 절차 실패는, 방문자가 자신의 세계로부터 밀려나는 과정으로 취급한다.

플레이어는 실패를 반복할수록 공간보다 먼저 "자기 인식"이 무너지는 압박을 체감한다.

### 적용 방식

- 퇴장 절차 시도 실패가 누적될 때마다 단계가 1씩 상승한다.
- 단계는 최대 3단계까지 적용한다.

### 단계 정의

1. **1단계: 공간 불안정화**
    - 복도 반복, 문 위치 어긋남, 환각 빈도 증가를 노출한다.
    - 연출은 공간 중심으로, 인물 기억은 아직 유지된다.
2. **2단계: 자아 흔들림**
    - 이름/목적/동행자 인식이 부분적으로 흔들리는 사건을 발생시킨다.
    - 같은 선택지를 다시 보게 하거나, 기억 서술이 어긋나는 방식으로 표현한다.
3. **3단계: 귀속 붕괴 직전**
    - 방문자가 "여기에서 사라지는" 감각을 직접 자각하게 한다.
    - 버려진 이 전이 위험을 명시하고, 회복 가능한 선택지를 제한한다.

### 구현 기준

- 실패 카운트 정수값(`exit_fail_count`)으로 단계 판정.
- 단계 변화 시 즉시 시각/대사/사운드 피드백 1회 이상 출력.

## 2) 정식 스펙 B — NORMAL/TRUE 엔딩 분기 고정

엔딩은 감정 해석이 아니라, 플레이어가 끝까지 어떤 구원을 선택했는지로 판정한다.

핵심 축은 네임드 귀신 성불 수다.

### 분기 기준

- 대상 네임드: **흉내내기 / 화재 / 추락 / 상대적 시간** (총 4체)
- **NORMAL**: 성불 0~1체
- **TRUE**: 성불 2~4체

### 성불 인정 조건

- 단순 제압/회피는 불인정.
- 해당 귀신의 "핵심 미련 해소 조건" 충족 시에만 인정.
- 한 귀신은 1회만 집계.

### 집계 시점

- 최종 퇴장 절차 직전 1회 고정 집계.

### 보완 해석

- TRUE는 완전한 구원 엔딩으로 고정한다.
- 세부 구원 구조와 조건은 데일리 작업본에서 별도 관리한다.

### 구현 기준

- 귀신별 Boolean 상태(`released_named_ghost_x`)를 저장.
- 집계 결과를 엔딩 분기 플래그로 즉시 변환.
- TRUE 진입 조건은 관련 상세 문서와 일치해야 한다.

## 4) 정식 스펙 D — 여권 프로토콜 예외 규정

여권은 세계 귀속의 임시 증명으로 간주한다.

이 문서는 "예외 상황에서도 규칙이 무너지지 않게" 하는 안전장치다.

### 기본 원칙

- 방문자 1인당 유효 여권 1개만 허용.
- 상태 흐름: 발급 → 검증 → 유지/파기/양도.

### 예외 규정

1. **분실**
    - 즉시 무효 처리.
    - 관리실 대면 검증 후 재발급 가능.
2. **훼손**
    - 핵심 식별면 훼손 시 검증 실패.
    - 재발급 시 기존 여권 자동 파기.
3. **위조 시도**
    - 즉시 무효.
    - 적발 시 퇴장 절차 실패 누적 +1.
4. **이중 소유 시도**
    - 신규 발급 순간 기존 여권 자동 소멸.
    - 동시 유효 상태는 허용하지 않음.
5. **양도 예외**
    - 방문자 → 버려진 이 단방향만 허용.
    - 양도 완료 시 원 소유자 방문자 자격 즉시 상실.

### 구현 기준

- 발급/재발급/양도/파기 이벤트를 모두 로그로 남김.
- 엔딩 판정 전 마지막 유효 상태를 최종 참조값으로 사용.

## 5) 정식 스펙 E — 퇴장 절차 구조 고정

퇴장 절차는 관리실에서 시작해, 처음 들어온 입구를 통해 종료되는 3단계 의식·검문 구조로 고정한다.

### 핵심 원칙

- 퇴장 절차는 여권, 장소, 선행 조건이 충족되었을 때만 개시 가능.
- 절차는 **개시 승인 → 귀속 검문 → 동반 확인 및 퇴장**의 3단계로 구성.
- 귀신은 절차 진행 중 직접 개입하지 않으며, 사전에 저지해야 할 위협으로 취급.
- 마지막 단계에서는 동행자 선언과 이름 확인이 필수.

### 구현 기준

- 시작 지점은 관리실, 종료 지점은 최초 입구로 고정.
- 절차 단계 수는 3단계 고정.
- 귀신의 절차 중 난입 이벤트는 두지 않는다.
- 세부 검문 항목과 TRUE 분기 조건은 데일리 작업본에서 상세 관리한다.

## 6) 정식 스펙 F — NORMAL 엔딩 구조 고정

NORMAL 엔딩은 임오준 개인의 퇴장만 성립하는 불완전한 생환 엔딩으로 고정한다.

### 핵심 원칙

- 임오준은 현실로 생환한다.
- 석수진의 세계 귀속 회복은 성립하지 않는다.
- 석수진은 백회빌라에 남는다.
- 임오준은 현실 복귀 후 석수진을 기억하지 못한다.
- 그에게는 "이상한 폐빌라에서 무언가 겪은 것 같다" 정도의 잔상만 남는다.

### 구현 기준

- NORMAL에서는 임오준 개인 퇴장만 성공 처리한다.
- 석수진 귀속 복원은 실패 상태로 유지한다.
- 엔딩 후 기억 연출은 불분명한 잔상 수준으로 제한한다.
- 세부 기억 처리와 감정선 대비는 데일리 작업본에서 상세 관리한다.

---

## 7) 정식 스펙 G — 관리자 권한 한계 고정

관리자는 백회빌라의 규칙을 집행하고 유지하는 존재이지, 세계 귀속을 임의로 재작성하는 존재는 아니다.

### 핵심 원칙

- 관리자는 여권 발급/검증, 퇴장 절차 승인, 상태 판정, 규칙 안내를 수행한다.
- 관리자는 공간 질서를 유지할 수 있으나, 없는 자격을 새로 만들 수는 없다.
- 버려진 이를 단독으로 현실에 내보내거나, 세계 귀속을 임의 복원할 수 없다.
- 귀신을 직접 해결하거나 엔딩 조건을 대신 충족시킬 수 없다.
- 연민은 가질 수 있으나, 예외를 만들어 규칙을 꺾지는 않는다.

### 구현 기준

- 관리자는 규칙 안내자이자 판정자로 연출한다.
- 관리자 대사에서는 감정보다 절차와 조건 확인이 우선되어야 한다.
- 버려진 이 직접 구제 불가 사유는 명시 가능해야 한다.
- 세부 권한 해설과 감정선 운용은 데일리 작업본에서 상세 관리한다.

## 9) 현재 합의 요약

- 퇴장 실패는 3단계 붕괴 구조로 확정한다.
- 엔딩은 네임드 귀신 성불 수 기반으로 확정한다.
- TRUE는 완전한 구원 엔딩으로 운영한다.
- NORMAL은 임오준만 생환하고 석수진을 기억하지 못하는 불완전한 생환 엔딩으로 운영한다.
- 퇴장 절차는 관리실 시작 / 최초 입구 종료 / 3단계 의식·검문 구조로 운영한다.
- 관리자는 규칙 집행자이며, 구원을 임의로 만들어내지는 못한다.
- 입장 트리거와 여권 예외는 서사-시스템 겸용 규칙으로 운영한다.
- 세부 감정선과 세계 귀속 회복 조건은 데일리 작업본에서 상세 관리한다.
- 문장은 서사 톤을 유지하되, 구현 가능 판정 기준을 반드시 포함한다.
